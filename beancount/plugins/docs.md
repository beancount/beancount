# Beancount Plugins

This directory contains a collection of optional plugins for Beancount. These plugins are Python modules that can be used to filter transactions, enforce constraints, validate data, or automate tasks within the Beancount ledger processing pipeline.

## Purpose

The plugins in this directory serve as:
-   **Examples:** Demonstrating how to write plugins for Beancount.
-   **Experiments:** Testing new ideas or constraints (e.g., `pedantic` checks).
-   **Utilities:** Providing useful functionality that is not part of the core (e.g., `auto_accounts`).

## Structure

Each plugin is typically a standalone Python module. To be recognized as a plugin, a module must define a `__plugins__` variable, which is a tuple of function names to be registered.

The standard signature for a plugin function is:

```python
def plugin_function(entries, options_map):
    """
    Args:
      entries: A list of directives (Transaction, Open, Close, etc.).
      options_map: A dictionary of parser options.
    Returns:
      A tuple (entries, errors), where:
      - entries: The modified list of directives.
      - errors: A list of new errors generated by the plugin.
    """
```

## Categories

The plugins can be broadly categorized into:

### 1. Automation & Generation
Plugins that modify the entry list to add missing information or automate boilerplate.
-   `auto_accounts.py`: Automatically inserts `Open` directives for accounts that are used but not explicitly opened.
-   `implicit_prices.py`: Synthesizes `Price` directives from `Transaction` postings that imply a price (e.g., conversions or explicit costs).

### 2. Validation & Constraints
Plugins that enforce specific rules or constraints on the ledger data.
-   `check_average_cost.py`: Validates that reducing legs on unbooked postings are close to the average cost basis (useful for `NONE` booking method).
-   `sellgains.py`: Verifies that declared gains match the prices on lot sales.
-   `leafonly.py`: Ensures that transactions are only posted to leaf accounts (accounts with no children).
-   `nounused.py`: Checks for accounts that are declared `Open` but never used.
-   `check_commodity.py`, `check_drained.py`, `check_closing.py`: Various other consistency checks.

### 3. Meta-Plugins
Plugins that aggregate other plugins for convenience.
-   `pedantic.py`: A "meta-plugin" that activates a suite of strict validation plugins (including `check_commodity`, `leafonly`, `nounused`, `sellgains`, etc.) to enforce a rigorous accounting style.

## Usage

Plugins are typically enabled in a Beancount input file using the `plugin` directive:

```beancount
plugin "beancount.plugins.auto_accounts"
plugin "beancount.plugins.pedantic"
```
