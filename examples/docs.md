# Beancount Examples

This directory contains a collection of example files and data illustrating the usage of Beancount, a double-entry bookkeeping computer language. These examples range from simple starter templates to complex, real-world scenarios and regression testing artifacts.

## Directory Structure

### `simple/`
Contains introductory examples designed for new users.
- **`basic.beancount`**: A comprehensive demonstration of Beancount syntax, including multi-currency transactions, various account types, and commodities.
- **`starter.beancount`**: A clean template intended to be copied and customized for starting a new personal ledger.

### `tutorial/`
Houses the output files generated by the official Beancount tutorial. These files represent the state of a ledger at various stages of the tutorial, showing the progression of a user's financial data.

### `vesting/`
Provides a specialized example for tracking Restricted Stock Units (RSUs).
- **`vesting.beancount`**: Demonstrates how to handle the grant, vesting, and taxation of RSUs by separating unvested shares (tracking commodity) from vested shares (liquid assets).

### `ingest/`
Contains artifacts and data structures used for testing the Beancount ingestion framework.
- **`office/`**: Organized by category (e.g., office-related documents), this directory includes "golden" files for regression testing of document importers, such as extracted text and expected account filings.

## Key Files

- **`example.beancount`**: A large, complex input file generated by the `bean-doctor` script. It serves as a stress test and a showcase of a fully populated ledger with extensive history and varied transaction types.

## Main Concepts

The examples in this directory collectively demonstrate:

1.  **Syntax & Structure**: Correct usage of directives like `open`, `pad`, `balance`, and transaction entries.
2.  **Commodity Handling**: Managing multiple currencies, stock tickers, and custom commodities (e.g., for unvested shares).
3.  **Account Hierarchy**: Best practices for organizing assets, liabilities, income, and expenses.
4.  **Ingestion & Testing**: How to structure data for verifying the correctness of document importers and text extraction.

## Usage

- **For Learning**: Browse `simple/basic.beancount` to understand how to record specific financial events.
- **For Starting**: Use `simple/starter.beancount` as a foundation for your own ledger.
- **For Advanced Scenarios**: Refer to `vesting/` for complex tracking needs or `ingest/` for understanding the import testing workflow.
